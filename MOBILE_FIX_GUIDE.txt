‚ùå PROBLEM: Getting 404 NOT_FOUND errors on mobile
‚úÖ SOLUTION: Your Windows Firewall is blocking mobile connections

üîß METHOD 1: Add Firewall Rule (RECOMMENDED)
1. Right-click on PowerShell and select "Run as Administrator"
2. Run this command:
   netsh advfirewall firewall add rule name="Django Dev Server" dir=in action=allow protocol=TCP localport=8000

üîß METHOD 2: Temporarily Disable Firewall (QUICK TEST)
1. Open Windows Settings
2. Go to Update & Security > Windows Security > Firewall & network protection
3. Turn off "Domain network", "Private network", and "Public network" firewalls
4. Test your mobile access
5. REMEMBER TO TURN THEM BACK ON LATER!

üîß METHOD 3: Use Different Port
Instead of port 8000, try port 80:
   python manage.py runserver 0.0.0.0:80

Then access: http://192.168.0.101/ (no :8000 needed)

üîß METHOD 4: Check Router Settings
Some routers have "AP Isolation" enabled which blocks device-to-device communication:
1. Access your router admin panel (usually 192.168.1.1 or 192.168.0.1)
2. Look for "AP Isolation" or "Client Isolation" 
3. Disable it if enabled

üì± TEST STEPS:
1. After applying any method above, test on your computer first:
   - Open browser and go to http://192.168.0.101:8000/
   - Should show your website (not 404)

2. Then test on mobile:
   - Make sure both devices are on same WiFi
   - Open mobile browser and go to http://192.168.0.101:8000/
